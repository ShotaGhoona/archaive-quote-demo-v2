# 03. 画面仕様

全ページ + ASCII モックアップ + コンポーネント分解 + 振る舞い。

技術スタック前提: Vite + React + Tailwind + shadcn/ui + React Router + React Query。実装パターンは poc-template の `docs/plan/loverbleでの実装方式.md` に従う。

---

## 1. ルーティング全体

```
/auth                                       公開: ログイン
/                                           ホーム → /items にリダイレクト
/items                                      Item 一覧
/items/:itemId                              Item 詳細 + 見積もりパネル ★中核
/setting/estimate-master                    見積もり設定トップ
/setting/estimate-master/categories/:name   カテゴリ別設定
/setting/estimate-master/lookup-tables      LOOKUP 一覧
/setting/estimate-master/lookup-tables/:id  LOOKUP 詳細
/setting/estimate-master/variables          変数定義
/setting/estimate-master/customers          取引先
/setting/estimate-master/units              単位
/setting/estimate-master/quote-formula      合計式設定
/setting/estimate-master/node-masters       ノードマスタ（後述）
```

すべて `<ProtectedRoute>` で囲む（認証は最低限、Supabase Auth の Email/Password）。

## 2. 画面 1: `/items` — Item 一覧

### 目的

過去案件・現案件の Item 一覧を表示。新規作成と過去案件のコピーの起点。

### モックアップ

```
┌─────────────────────────────────────────────────────────────┐
│ Header: ロゴ / Items / 設定 / ユーザーアイコン                 │
├─────────────────────────────────────────────────────────────┤
│  Item 一覧                              [+ 新規 Item]        │
│  検索: [               ]  ステータス: [全て ▼]                │
│ ─────────────────────────────────────                      │
│  品番       | 品名              | 取引先      | 更新日 |     │
│  ────────────────────────────────────                      │
│  ABC-001   | シャフト A-100    | ★★製作所   | 04/28 |[→][複製]│
│  ABC-002   | フランジ B-50     | ◯◯工業     | 04/27 |[→][複製]│
│  ...                                                        │
└─────────────────────────────────────────────────────────────┘
```

### 主要コンポーネント

```tsx
<AppLayout>
  <PageHeader title="Items" action={<Button>+ 新規 Item</Button>} />
  <SearchBar />
  <Table columns={[品番, 品名, 取引先, 更新日, アクション]} />
</AppLayout>
```

### 振る舞い

- 行クリック → `/items/:itemId` へ
- `[複製]` → 既存 Item をコピーして新規作成（過去案件流用）
- `[+ 新規]` → モーダルで品番・品名・取引先を入力
- 検索・フィルタ・ソートは React Query データを `useMemo` で絞る

### 使うフック

- `useItems()` — 一覧取得
- `useCreateItem()` — 新規作成
- `useDuplicateItem()` — 複製（明細・変数値ごとコピー）

### CategoryTab の中身

```tsx
<CategoryTab category="MATERIAL">
  {detailRows.map(row => (
    <DetailRow key={row.id}>
      <HierarchySelector
        tree={selectionTree[category]}
        path={row.path}              // 例: ["板金材", "SS400", "重量ベース"]
        onPathChange={updatePath}
      />
      {row.formulaTemplateId && (
        <VariableInputArea
          template={row.template}
          values={row.variableValues}
          lockedVariables={row.derivedFromPath} // パス変数はロック表示
          onChange={updateVariable}
        />
      )}
      <SubtotalLabel
        value={row.subtotal}
        onClickPen={() => openAdjustModal('subtotal', row.id)}
      />
    </DetailRow>
  ))}
  <Button onClick={addNewRow}>+ 追加</Button>
</CategoryTab>
```

### HierarchySelector の振る舞い

- 1 階層目のプルダウンを表示
- 選択 → そのノードに子があれば 2 階層目を表示
- 葉に到達した瞬間に formula_template_id が確定
- 各ノードの **B 役割（マスタ参照）** で master_id を確定 → ノード以降の階層で MASTER 変数が利用可能に
- 各ノードの **C 役割（固定変数）** で fixed_variables をパスに追加

### VariableInputArea の振る舞い

- 計算式テンプレートが要求する変数を Form で出す
- 変数のソースタイプによって UI 切替:
  - MANUAL: 通常の Input
  - MASTER（パスから取得済み）: ロック表示（🔒 + 値）
  - LOOKUP: ロック表示（🔒 + 値、ホバーで「LOOKUP "XX 表" から取得」）
- 値変更でデバウンス 200ms → サーバー再計算 → 小計が更新

### 使うフック

- `useItem(itemId)` — Item 取得
- `useActiveQuote(itemId)` — 編集中の Quote
- `useUpdateQuote()` — Quote 更新（明細・変数値含む）
- `useRecalculate()` — サーバー再計算（保存時 + 計算プレビュー）
- `useSelectionTree(category)` — 選択肢の木
- `useFormulaTemplate(id)` — 計算式テンプレート

## 6. 画面 5: `/setting/estimate-master/node-masters` — ノードマスタ

### 目的

選択肢の木の各階層で使う「種類」を定義する。

例:
- 「材料項目」（板金材／角材／鋳造材）
- 「材料種別」（SS400／SUS304／A5052）
- 「計算ベース」（重量ベース／体積ベース）
- 「加工方法」（フライス／旋盤／メッキ）
- 「取引先ランク」（A／B／C）

### モックアップ

```
┌──────────────────────────────────────────────────────────────────┐
│ ノードマスタ                                       [+ 新規]        │
├──────────────────────────────────────────────────────────────────┤
│ 種類名         | 持つ属性                | 紐づく数 | アクション      │
│ ───────────────────────────────────                              │
│ 材料項目       | title                   | 3        | [✏][🗑]      │
│ 材料種別       | title, category, 比重,  | 4        | [✏][🗑]      │
│              | 単価                    |          |              │
│ 計算ベース     | title                   | 2        | [✏][🗑]      │
│ 加工方法       | title                   | 5        | [✏][🗑]      │
│ 取引先         | title, ランク, 連絡先   | 12       | [✏][🗑]      │
└──────────────────────────────────────────────────────────────────┘
```

### ノードマスタ詳細モーダル

```
┌────────────────────────────────────────┐
│ ノードマスタ: 材料種別                  │
├────────────────────────────────────────┤
│ 種類名 [材料種別           ]            │
│                                        │
│ 持つ属性                                │
│  ┌─────────────────────────────────┐  │

│  │ [+ 属性追加]                       │  │
│  └─────────────────────────────────┘  │
│                                        │
│ 登録された値                            │
│  ┌─────────────────────────────────┐  │
│  │ title    | category | 比重 | 単価 │  │
│  │ SS400    | 鉄鋼     | 7.85 | 380 │  │
│  │ SUS304   | ステンレス| 7.93 | 420 │  │
│  │ A5052    | アルミ   | 2.68 | 600 │  │
│  │ [+ 値を追加]                       │  │
│  └─────────────────────────────────┘  │
│                                        │
│         [キャンセル]  [保存]            │
└────────────────────────────────────────┘
```

### この画面の意義

選択肢の木のノードは、まず「種類」（ノードマスタ）を選び、次にその種類の中の「具体値」を選ぶ、という 2 段階モデル。

→ ノード階層編集時、各階層で **どのノードマスタを使うか** を指定できる。例: 階層 1 = 材料項目、階層 2 = 材料種別、階層 3 = 計算ベース。

```
│ 軸: 中計合計（=原価合計）                                         │
│                                                                  │
│ 丸め:                                                            │
│ ・方式 ○ FLOOR  ● CEIL  ○ ROUND                                  │
│ ・桁数 [-2     ]  (例: -2 = 100 円単位)                          │
│                                                                  │
│ H の式プレビュー:                                                │
│ 合計 = 丸め( Σ 中計 × (1 + 利益率表[Σ 中計]), -2 )                │
└──────────────────────────────────────────────────────────────────┘
```

## 10. 共通レイアウト

### `<AppLayout>`

- 左サイドバー: ロゴ / Items / 設定 / ユーザー
- 上部: パンくず or タイトル
- メイン: `<Outlet />`

### `<SettingsLayout>`

- 左サイドバー: 設定の各セクション
- メイン: `<Outlet />`（子ルート）

### `<PageHeader>`

shadcn の Card と Button を組み合わせた共通ヘッダ。

## 11. 状態管理（Zustand）

シンプルに、画面ローカルなものは `useState`、画面横断は React Query キャッシュ。

例外的に Zustand を使うべきもの:
- `useQuoteEditStore` — Item 詳細画面で編集中の見積もり全体（明細・変数値・上書き）

それ以外（マスタ系の編集など）は React Query + サーバー保存で十分。
