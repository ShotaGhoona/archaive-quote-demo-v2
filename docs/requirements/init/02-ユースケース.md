# 02. ユースケース

ペルソナ × 業務シナリオで「どう使うか」を描く。実装はこれを動かせる状態を目指す。

---

## ペルソナ

### P1. 営業担当 — 佐藤さん（50 代）

電話と FAX が主戦場のベテラン営業。図面を見て「この前の案件と似てる」と判断するのが得意。詳しい計算式は分からないが、「過去案件と同じくらいの値段にしたい」と思っている。

**主な操作**: 過去の見積もり一覧から類似品をコピー、数量を変えて再見積もり、PDF で顧客に提出。

### P2. 設計・積算担当 — 田中さん（30 代）

社内の原価責任者。計算式やマスタ運用を保守する。複雑な形状の材料費を体積計算から積み上げるのが日常業務。

**主な操作**: 計算式テンプレートの追加・編集、LOOKUP テーブルの行データ更新、複雑な見積もりの作成。

### P3. 管理者 — 鈴木さん（40 代）

経理兼システム管理者。単価更新と監査対応が仕事。「この見積もりの根拠を出せ」と言われたら出せる状態を保つ。

**主な操作**: 材料単価・時間単価の更新、利益率モードの切替、過去見積もりの再現性チェック。

## ユースケース

### UC-1. 標準的な見積もり（佐藤さん）

**状況**: 顧客から「SUS304 丸棒 φ50×200mm を 50 本」と電話。

**手順**:
1. Item 画面を開く（既存 Item の場合）
2. 見積もりパネルで「材料」タブを選ぶ
3. [+ 追加]
4. プルダウン: 「板金材」→「SS400」→「重量ベース」を選択
5. 葉に到達 → 計算式が確定し、変数入力エリアが現れる
6. 板厚・長さを入力（直径と単価はマスタから自動セット、ロックアイコン表示）
7. 小計が即時表示
8. 「工程」タブに切替、旋盤加工 30 分を入力
9. 全部入れ終わったら、合計エリアで売価を確認
10. ペンマークで合計を 6,500 円に切り上げ調整
11. 保存

**システムが内部でやること**:
- マスタから比重・単価を取得
- LOOKUP「歩留まり表」から SS400 / φ50 の係数を取得
- F の計算式を評価して小計
- G_pass で中計
- H_overall で合計に利益率を掛けて丸め

---

### UC-2. 段階見積もり（佐藤さん）

**状況**: 顧客から「とりあえず 100 個でお願いしたいけど、500 個ならいくら？1000 個なら？」

**手順**:
1. Item 画面で見積もりを作成
2. 数量バリエーションタブで [100] [500] [1000] を作成
3. 共通の明細を入力（材料・加工）
4. 各バリエーションで自動的に再計算される（数量が変数として変わるため）
5. 段取り費は固定なので 1 個あたり単価が下がる
6. 利益率 LOOKUP（数量軸）が設定されていれば、数量ごとに率も変わる
7. PDF 出力で 3 つの数量を並列表示

**期待される出力**:

| 数量 | 合計 | 1 個あたり |
|---|---|---|
| 100 | 138,000 円 | 1,380 円 |
| 500 | 650,000 円 | 1,300 円 |
| 1000 | 1,200,000 円 | 1,200 円 |

---

### UC-3. 計算式テンプレートの追加（田中さん）

**状況**: 新しい材料「真鍮丸棒」に対応したい。比重と単価が違うので新しい計算式が要る。

**手順**:
1. 設定画面 → 計算式テンプレート → [+ 新規]
2. 名前: 「真鍮丸棒・重量計算」
3. 対象: 材料費
4. 式: `(d/2)² × π × L × 比重/10⁶ × 単価 × 歩留まり`
5. 使用変数: `d, L` を MANUAL、`比重, 単価, 歩留まり` を MASTER 参照（材料マスタから）
6. テスト計算で動作確認
7. 保存

その後、設定画面 → 材料費の選択肢の木 → 「角材」→「真鍮」→「重量ベース」を追加し、上で作った計算式を紐付け。

---

### UC-4. LOOKUP テーブルの更新（田中さん）

**状況**: 焼入れ料金表の値が変わった（1 年ごとに見直し）。

**手順**:
1. 設定画面 → LOOKUP → 「焼入れ料金表」
2. [新バージョン作成] ボタン → 旧版が ARCHIVED に
3. 新バージョンで行データを編集
4. CSV インポートで一括更新も可
5. 保存
6. 新規見積もりは新バージョンで動く
7. 過去見積もりは旧バージョンで再現される（再計算しても同じ結果）

---

### UC-5. 過去見積もりの再現性チェック（鈴木さん）

**状況**: 監査で「2 ヶ月前のあの見積もり、根拠を見せろ」と言われた。

**手順**:
1. 見積もり一覧で該当を検索
2. 開く → 保存時点の値（中計・合計）が表示される
3. 「再計算」ボタンで現マスタを使った場合の値と並列表示
4. マスタが変わっていれば差分が見える
5. PDF 出力で根拠を含めて提出

**システムが内部でやること**:
- 保存時の `formula_template_version_id`, `material_version_id`, `lookup_version_ids` を使って再計算
- 結果は当時と同じ

---

### UC-6. 利益率モードの切替（鈴木さん）

**状況**: 営業会議で「これからは原価合計が大きい案件は利益率を下げる方針」と決まった。

**手順**:
1. 設定画面 → 合計式設定
2. 利益率モード: 「LOOKUP」を選択
3. 利益率テーブル: 「全体利益率表」を選択（事前に作成済み）
4. 軸: 原価合計（中計の合計）
5. プレビューで H の式が更新される
6. 保存
7. 新規見積もりから新方式が適用される

LOOKUP 表の例:
```
[0, 5万)        → 30%
[5万, 50万)     → 25% → 15% (LINEAR 補間)
[50万, ∞)       → 10%
```

---

### UC-7. 上書き 3 レバーの活用（佐藤さん）

**状況**: 計算結果の合計が 6,367 円。顧客との関係で「6,500 円ぴったりにしたい」。

**選択肢 A: 完全上書き**
- ペンマーク → 完全上書き → `6500`
- 採用値 = 6,500

**選択肢 B: 倍率調整**
- ペンマーク → 倍率 → `× 1.021` （計算: 6500 / 6367 ≈ 1.021）
- 採用値 = 6,367 × 1.021 = 6,500

**選択肢 C: 加算調整**
- ペンマーク → 加算 → `+ 133`
- 採用値 = 6,367 + 133 = 6,500

どれを選ぶかは履歴に残る → 後から「なぜこの値になったか」を追跡できる。

---

### UC-8. 取引先選択で変数注入（田中さん）

**状況**: メッキを ◯◯メッキ工業と △△鉱業のどちらに出すかで単価が違う。

**選択肢の木の設計**:

```
外注費
└─ メッキ
    ├─ ◯◯メッキ工業 (CUSTOMER 参照)
    │   ├─ 通常 (固定変数: 係数=1.0)
    │   └─ 特急 (固定変数: 係数=1.5)
    └─ △△鉱業 (CUSTOMER 参照)
        └─ 通常 (固定変数: 係数=1.0)
```

**手順**:
1. 「メッキ」→「◯◯メッキ工業」→「特急」を選ぶ
2. 葉に到達: 計算式 `面積 × 単価 × 係数 + 運賃`
3. 取引先のランクから単価が LOOKUP で取得される
4. 「特急」ノードの固定変数で係数 = 1.5 が注入される
5. 面積と運賃をユーザーが入力 → 小計

---

### UC-9. 諸経費の手入力（佐藤さん）

**状況**: 梱包費 500 円を計上したい。計算式は要らない、項目名と金額だけ。

**手順**:
1. 「諸経費」タブ → [+ 追加]
2. プルダウン: 「汎用諸経費」を選ぶ（諸経費のテンプレは 1 つだけ）
3. 葉に到達: 計算式 `金額`（= 入力された金額そのまま）
4. 変数入力エリア: 項目名「梱包費」、金額「500」
5. 小計 = 500 円

→ **緑パターン（手入力）も統一パターンの特殊形** として吸収。

---

### UC-10. 見積もりコピーで類似案件の流用（佐藤さん）

**状況**: 半年前のシャフト案件と似た図面が来た。

**手順**:
1. Item 一覧で過去案件を検索
2. 開く → [コピー] ボタン
3. 新規 Item として複製（明細・変数値もコピー）
4. 品名と数量だけ変える
5. 自動再計算で新しい合計が出る
6. 保存して提出
