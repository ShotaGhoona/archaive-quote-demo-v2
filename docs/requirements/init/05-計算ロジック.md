# 05. 計算ロジック

サーバーサイドで実装する計算アルゴリズムの厳密な仕様。フロントエンドは UX のための速報計算のみ行い、保存時には必ずサーバー再計算する。

用語は [01-用語と概念モデル.md](./01-用語と概念モデル.md) に従う。

---

## 1. 全体の式

```
合計 = 丸め( adjust_H( H( [adjust_G(G_c(...)) for c in categories] ) ) )

中計_c = adjust_G( G_c( [adjust_F(F(...)) for 明細 in c] ) )

小計   = adjust_F( F( 変数達 ) )
```

ここで `adjust(x)` は上書き 3 レバー:

```
adjust(x) = もし override_value が設定されていれば → override_value
            そうでなければ                       → x × multiplier + adjustment
```

## 2. 変数解決の流れ

明細 1 件に対して、計算式の評価に必要な変数を集める。

### 2.1 入力情報

- `selection_path`: 葉に至るまでのノード ID の配列
- `formula_template_id`: 葉が指す計算式テンプレート
- `variable_values`: ユーザーが入力した値（手入力 = MANUAL ソースの変数のみ）

### 2.2 計算式テンプレートが要求する変数の取得

```python
def resolve_variables(detail):
    template = get_formula_template(detail.formula_template_id)
    used_var_codes = parse_variables_from_formula(template.formula)
    # 例: '(d/2)^2 * PI * L * rho/1000000 * P * Y' → ['d', 'L', 'rho', 'P', 'Y']

    # パス上のノードマスタ値の属性を順番にマージ（深い方が勝つ）
    path_attrs = {}
    for node_id in detail.selection_path:
        node = get_node(node_id)
        if node.node_master_value_id:
            value = get_node_master_value(node.node_master_value_id)
            # value.values は { "title": "SS400", "density": 7.85, "unit_price": 380, ... }
            for k, v in value.values.items():
                path_attrs[k] = v   # 後勝ち

    context = {}

    # 各変数を解決
    for var_code in used_var_codes:
        var_def = get_variable_definition_by_code(var_code)

        if var_def.source == 'MANUAL':
            if var_code in detail.variable_values:
                context[var_code] = detail.variable_values[var_code]
            elif var_def.default_value is not None:
                context[var_code] = var_def.default_value
            elif var_def.required:
                raise UndefinedVariableError(var_code)

        elif var_def.source == 'PATH':
            # path_key で経路属性から取り出す
            key = var_def.path_key or var_code
            if key in path_attrs:
                context[var_code] = path_attrs[key]
            elif var_def.default_value is not None:
                context[var_code] = var_def.default_value
            elif var_def.required:
                raise UndefinedVariableError(var_code)

        elif var_def.source == 'LOOKUP':
            # LOOKUP の軸値を context（+ path_attrs）から集めて検索
            lookup_table = get_lookup_table(var_def.lookup_table_id)
            axis_values = [
                context.get(axis['variable_code']) or path_attrs.get(axis['variable_code'])
                for axis in lookup_table.axes
            ]
            value, version_id = lookup_search(lookup_table, axis_values)
            context[var_code] = value
            # 使用バージョン記録
            detail.lookup_versions_used.append({
                'lookup_table_id': lookup_table.id,
                'version_id': version_id
            })

    return context
```

旧設計の `MASTER` ソース（`source_entity` + `source_field` で材料/工程/取引先テーブルを直接引く）は削除し、PATH に統合した。

## 3. F: 計算式評価（AST パーサー）

`eval()` は使わず、AST パーサーで安全に評価する。実装では `mathjs` を採用しているが、要件としては「式を文字列として保存し、評価時に変数を bind する」ことが本質。以下は概念コード（Python 風）。

```python
import ast
import math

SAFE_FUNCTIONS = {
    'round': round,
    'ceil': math.ceil,
    'floor': math.floor,
    'max': max,
    'min': min,
    'abs': abs,
    'sqrt': math.sqrt,
}

CONSTANTS = {
    'PI': math.pi,
}

def evaluate_formula(formula: str, context: dict, lookup_func=None) -> float:
    # '^' を '**' に変換（文字列リテラル内は除外）
    processed = preprocess_formula(formula)
    tree = ast.parse(processed, mode='eval')
    return _eval(tree.body, context, lookup_func)

def _eval(node, ctx, lookup):
    if isinstance(node, ast.Constant):
        return float(node.value) if isinstance(node.value, (int, float)) else node.value

    if isinstance(node, ast.Name):
        if node.id in CONSTANTS:
            return CONSTANTS[node.id]
        if node.id in ctx:
            v = ctx[node.id]
            return float(v) if isinstance(v, (int, float)) else v
        raise UndefinedVariableError(node.id)

    if isinstance(node, ast.Attribute):
        # material.比重 のようなドット名
        dotted = f"{node.value.id}.{node.attr}"
        if dotted in ctx:
            return float(ctx[dotted])
        raise UndefinedVariableError(dotted)

    if isinstance(node, ast.BinOp):
        L = _eval(node.left, ctx, lookup)
        R = _eval(node.right, ctx, lookup)
        op_type = type(node.op)
        if op_type is ast.Div and R == 0:
            raise DivisionByZeroError()
        result = OPERATORS[op_type](L, R)
        if abs(result) > 1e15:
            raise OverflowError()
        return float(result)

    if isinstance(node, ast.UnaryOp):
        return -_eval(node.operand, ctx, lookup)

    if isinstance(node, ast.Call):
        func_name = node.func.id
        args = [_eval(a, ctx, lookup) for a in node.args]
        if func_name == 'LOOKUP':
            return lookup(*args)
        if func_name in SAFE_FUNCTIONS:
            return float(SAFE_FUNCTIONS[func_name](*args))
        raise SyntaxError(f"Unknown function: {func_name}")

    raise SyntaxError(f"Unsupported: {type(node).__name__}")
```

### 3.1 サポートする式の文法

| 演算子・関数 | 用途 |
|---|---|
| `+ - * / %` | 四則と剰余 |
| `^` または `**` | べき乗 |
| `( )` | 括弧 |
| `round, ceil, floor, max, min, abs, sqrt` | 関数 |
| `PI` | 円周率定数 |
| `LOOKUP("表名", 軸値1, 軸値2, ...)` | 参照表検索 |

### 3.2 諸経費の「直接代入式」

諸経費の計算式テンプレートは `金額` という単一変数だけの式:

```python
formula = "金額"
context = {"金額": 500}
result = evaluate_formula(formula, context)  # → 500
```

→ 緑パターン（手入力）も統一パターンで吸収できる。

## 4. LOOKUP: 軸検索 + 線形補間

```python
def lookup_search(lookup_table, axis_values):
    active_version = get_active_version(lookup_table.id)
    rows = get_rows(active_version.id)

    matched = []
    for row in rows:
        if all(_match_axis(axis, row.conditions, value)
               for axis, value in zip(lookup_table.axes, axis_values)):
            matched.append(row)

    if len(matched) == 0:
        raise LookupNotFoundError()
    if len(matched) > 1:
        raise LookupAmbiguousError()

    row = matched[0]
    # 補間
    if row.interpolation == 'CONSTANT':
        return row.return_value, active_version.id

    if row.interpolation == 'LINEAR':
        # RANGE 軸を 1 本だけ持つ前提
        range_axis = next(a for a in lookup_table.axes if a['match_type'] == 'RANGE')
        x = axis_values[range_axis['axis_order'] - 1]
        x_min = row.conditions[f"{range_axis['variable_code']}_min"]
        x_max = row.conditions[f"{range_axis['variable_code']}_max"]
        v_min = row.return_value
        v_max = row.return_value_max
        ratio = (x - x_min) / (x_max - x_min)
        return v_min + (v_max - v_min) * ratio, active_version.id

def _match_axis(axis, conditions, value):
    code = axis['variable_code']
    if axis['match_type'] == 'EXACT':
        return str(conditions.get(code)) == str(value)
    if axis['match_type'] == 'RANGE':
        x_min = conditions.get(f"{code}_min")
        x_max = conditions.get(f"{code}_max")
        return x_min is not None and x_max is not None and float(x_min) <= float(value) < float(x_max)
    if axis['match_type'] == 'PREFIX':
        prefix = conditions.get(f"{code}_prefix") or conditions.get(code)
        return str(value).startswith(str(prefix))
```

## 5. G: 中計の計算

ビルトインテンプレートで分岐:

```python
def evaluate_g(category, details, params, transport_costs=None):
    g_template = get_g_template(category)

    if g_template.builtin_key == 'G_pass':
        return sum(d.final_value for d in details)

    if g_template.builtin_key == 'G_outsource':
        sum_subtotals = sum(d.final_value for d in details)
        sum_transport = sum(transport_costs or [])
        return sum_subtotals + sum_transport

    if g_template.builtin_key == 'G_margin':
        margin = params.get('category_margin', 0)
        return sum(d.final_value for d in details) * (1 + margin)

    # カスタムテンプレート（将来拡張）
    raise NotImplementedError()
```

## 6. H: 合計の計算

```python
def evaluate_h(intermediates, settings, lookup_func=None):
    sum_int = sum(intermediates.values())  # 中計の合計

    if settings.margin_mode == 'CATEGORY':
        # G_margin を各カテゴリで適用済みなので、ここでは合計するだけ
        return sum_int

    if settings.margin_mode == 'TOTAL':
        return sum_int * (1 + settings.total_margin)

    if settings.margin_mode == 'LOOKUP':
        rate = lookup_search(settings.total_margin_table, [sum_int])[0]
        return sum_int * (1 + rate)
```

### 6.1 循環防止

LOOKUP モードの軸として「合計（売価）」を選んだ場合、循環するためエラーにする。許可される軸:

- 中計の合計（= 原価合計）
- カテゴリ別の中計
- 数量

DB 設定時にバリデーション必須。

## 7. 丸め

```python
def apply_rounding(value, method, digits):
    factor = 10 ** digits
    if method == 'FLOOR':
        return math.floor(value * factor) / factor
    if method == 'CEIL':
        return math.ceil(value * factor) / factor
    if method == 'ROUND':
        return round(value * factor) / factor
```

例: `value=6477.83, method=ROUND, digits=-2` → `6500`

## 8. 上書き 3 レバー

```python
def adjust(computed, multiplier=1.0, adjustment=0.0, override=None):
    if override is not None:
        return override
    return computed * multiplier + adjustment
```

各層（小計・中計・合計）で同じ関数を使う。

## 9. 全体フロー: 1 つの数量バリエーションを計算する

```python
def compute_lot(quote_lot_id):
    lot = get_quote_lot(quote_lot_id)
    quote = get_quote(lot.quote_id)
    settings = get_quote_formula_settings()

    intermediates = {}

    for category in get_enabled_categories(settings):
        # 1. 各明細の小計
        details = get_details(lot.id, category)
        for d in details:
            ctx = resolve_variables(d)
            try:
                d.computed_value = evaluate_formula(
                    d.formula_template.formula,
                    ctx,
                    lookup_func=lambda *args: lookup_search(...)[0]
                )
            except Exception as e:
                d.computed_value = None
                d.error = str(e)
                continue
            d.final_value = adjust(d.computed_value, d.multiplier, d.adjustment, d.override_value)
            save(d)

        # 2. 中計
        cat_total = get_category_total(lot.id, category)
        cat_total.computed_value = evaluate_g(
            category, details, cat_total.g_params,
            transport_costs=[d.transport_cost for d in details if category == 'OUTSOURCE']
        )
        cat_total.final_value = adjust(cat_total.computed_value, cat_total.multiplier, cat_total.adjustment, cat_total.override_value)
        save(cat_total)
        intermediates[category] = cat_total.final_value

    # 3. 合計
    lot.computed_total = evaluate_h(intermediates, settings)
    final_before_round = adjust(lot.computed_total, quote.total_multiplier, quote.total_adjustment, quote.total_override)
    lot.final_total = apply_rounding(final_before_round, settings.rounding_method, settings.rounding_digits)

    # 4. 1 個あたり単価
    lot.unit_price = lot.final_total / lot.quantity

    save(lot)
```

## 11. スナップショット保存

見積もり保存時、再現性のために以下を固定する:

| 項目 | 保存場所 |
|---|---|
| 変数値 | `quote_details.variable_values` |
| LOOKUP バージョン | `quote_details.lookup_versions_used` |
| 利益率モードと値 | `quotes.margin_mode`, `quotes.total_margin` 等 |
| 丸め設定 | `quotes.rounding_method`, `quotes.rounding_digits` |
| 計算結果 | `quote_details.final_value`, `quote_category_totals.final_value`, `quote_lots.final_total` |

→ 過去見積もりを開いたとき、保存された値で再現される。再計算ボタンを押すと現マスタを使った値と並列表示。

## 12. エラー処理

| エラー | 発生条件 | UI 表示 |
|---|---|---|
| `UndefinedVariableError` | 必須変数が未定義 | 該当明細に赤バッジ「変数 XX が未入力」 |
| `LookupNotFoundError` | LOOKUP 該当行なし | 「条件に該当する値がありません」 |
| `LookupAmbiguousError` | LOOKUP 複数該当 | 「複数該当：表の設定を確認してください」 |
| `DivisionByZeroError` | ゼロ除算 | 「式にゼロ除算があります」 |
| `OverflowError` | `|x| > 1e15` | 「計算結果が大きすぎます」 |
| `SyntaxError` | 不正な計算式 | 「計算式の構文エラー」 |
| `MarginCycleError` | LOOKUP 軸に「合計」を指定 | 「利益率の軸に売価は使えません」 |

エラーが起きた明細は `final_value = NULL`、合計計算からは除外して **不正確な合計を表示しない**（=エラー解消が必要）。

## 13. パフォーマンス最適化

デモではほぼ不要だが念のため:

- 1 つの lot で 50 明細以下を想定
