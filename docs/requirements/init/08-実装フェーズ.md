# 08. 実装フェーズ

claude-code に渡すときの **作業の順序と完成定義**。Phase ごとに動作確認できる状態を作りながら進める。

各 Phase は前の Phase の上に積み上がる。1 つずつ完成させてから次に進む。

---

## Phase 0: セットアップ（半日）

### やること

1. `poc-template` を新しいディレクトリにクローン
2. `npm install` で依存解決
3. Supabase プロジェクト作成（or 既存プロジェクトの URL/Key を `.env` に）
4. このフォルダ（`v2-demo作成必要資料/`）を `docs/` 配下にコピー
5. 既存の `docs/plan/` も残す（実装パターンの参照用）
6. README.md を更新して `Archaive-Quote v2 デモ` と書き換え

### 完成定義

- `npm run dev` で `http://localhost:8080` が起動する
- Supabase Auth でログインできる（poc-template デフォルト機能）
- `docs/00-製品概要.md` 等が読める

## Phase A: DB 構築 + 認証（1〜2 日）

### やること

1. `04-DB設計.md` のマイグレーション 11 ファイルを `supabase/migrations/` に配置
2. `npx supabase db push` で適用
3. `07-初期データ.md` の seed SQL を実行
4. Supabase 型生成: `npx supabase gen types typescript --project-ref XXX > src/integrations/supabase/types.ts`
5. 認証は Email/Password のみ（poc-template の Auth 画面そのまま）

### 完成定義

- Supabase Studio で全テーブルが見える
- seed データが入っている（材料、工程、取引先、Item ABC-001 と ABC-002）
- ログイン後にダッシュボードに遷移する
- `useItems()` フックを書いて Items 一覧が API から取れる（コンソール確認 OK）

## Phase D: Item 詳細 + 見積もり作成（コア、3〜4 日）

### やること

1. `/items` Item 一覧ページ
2. `/items/[id]` Item 詳細ページのレイアウト（左: 図面、右: 見積もりパネル）
3. 図面プレビュー（Supabase Storage アップロード + 表示）
4. CategoryTabs（5 カテゴリ）
5. **HierarchySelector**（N 階層プルダウン、葉到達検出）
6. **VariableInputArea**（計算式が要求する変数の動的フォーム）
   - MASTER ソース変数はロック表示（🔒 アイコン）
   - LOOKUP ソース変数もロック表示
   - MANUAL のみ編集可
7. 小計のリアルタイム計算（デバウンス 200ms → サーバー API）
8. 中計と合計の表示
9. 数量バリエーションタブ（最初は単一バリエーションで OK）

### 完成定義

- ある Item で材料費 1 件 + 工程費 1 件を入力 → 合計が出る
- 選択肢の木を辿って葉に到達したとき、計算式が一意に決まる
- パス上のノードのマスタ参照で、変数が自動セット（ロック表示）
- 変数値を変えると小計がリアルタイム更新
- 5 カテゴリすべてで明細追加・編集できる
- 諸経費の手入力（項目名 + 金額）が動く

## Phase E: 上書き 3 レバー + 利益率モード（2 日）

### やること

1. ペンマーク UI を小計・中計・合計に配置
2. AdjustModal（3 レバー編集）の実装
3. `/setting/estimate-master/quote-formula` ページ
4. 利益率モード切替（CATEGORY / TOTAL / LOOKUP）
5. 各モードに応じて H と G のテンプレートが切り替わる

### 完成定義

- ペンマーク → モーダル → 倍率 1.1 を設定 → 採用値が反映される
- 完全上書きを設定したら計算結果を無視する
- 利益率モードを変えると合計の計算ロジックが切り替わる
- LOOKUP モードで「利益率表」を選ぶと、原価合計から利益率が引かれる

## トラブルシューティング Tips（claude-code 向け）

### 計算が合わない

1. F の式が AST パーサーで評価可能か確認（変数名が DB の variable_definitions と一致しているか）
2. パス変数の解決が動いているか console.log で確認
3. LOOKUP の軸値の型（文字列 vs 数値）に注意

### 選択肢の木が描画されない

1. `parent_id IS NULL` のノードを「ルート」として再帰的に描画
2. category でフィルタしているか確認
3. ドラッグ&ドロップは `@dnd-kit` を別途インストール

### 上書き 3 レバーの計算

`adjust(x) = override ?? (x * multiplier + adjustment)` の式は server side で 1 箇所に集約。フロントは値を渡すだけ。
