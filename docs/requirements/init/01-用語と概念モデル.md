# 01. 用語と概念モデル

このドキュメントは Archaive-Quote v2 の **共通言語** と **計算モデルの骨組み** を定義する。以降の資料はすべてこの用語に従う。

---

## 1. 金額の階層（合計 / 中計 / 小計）

```
合計           ← 見積もり全体の最終売価
  ├ 中計 (材料費)    ← 見積カテゴリごとの合計
  │   ├ 小計 (明細1)  ← 1 件の明細の金額
  │   ├ 小計 (明細2)
  │   └ ...
  ├ 中計 (工程費)
  ├ 中計 (外注費)
  ├ 中計 (購入品)
  └ 中計 (諸経費)
```

| 用語 | 意味 |
|---|---|
| **合計** | 見積もり全体の最終売価。利益率と丸めまで適用済み |
| **中計** | ある見積カテゴリの合計。複数の小計を集約したもの |
| **小計** | 1 件の明細の金額。変数入力を受けて計算式が出す数字 |

---

## 2. 見積カテゴリ（5 種固定）

```
見積カテゴリ ∈ { 材料費, 工程費, 外注費, 購入品, 諸経費 }
```

会社単位で **ON/OFF** できる（使わないカテゴリは表示しない）。順番もこの 5 つで固定。

---

## 3. 計算式の 3 層: F, G, H

```
F: 変数達       → 小計        (明細 1 件の式)
G_c: 小計達    → 中計         (見積カテゴリ c ごとの集約式)
H: 中計達      → 合計         (全体集約式 + 丸め)
```

### F（小計の式）

各明細の金額を出す。**自由設計の計算式テンプレート**:
- 材料費: `(d/2)² × π × L × 比重 × 単価 × 歩留まり`
- 工程費: `加工時間 × 時間単価 + 段取り費`
- 諸経費（手入力）: `金額` ← 直接代入

### G（中計の式）

ビルトインテンプレートから選ぶ（v2 では完全カスタムは不要）:

| テンプレ名 | 式 | 用途 |
|---|---|---|
| `G_pass` | `Σ 小計` | 普通の合計（中計で利益率を掛けない） |
| `G_outsource` | `Σ 小計 + Σ 運賃` | 外注費の運賃加算 |
| `G_margin` | `Σ 小計 × (1 + 利益率_c)` | カテゴリ別利益率モード時 |

### H（合計の式）

ビルトインテンプレートから選ぶ:

| テンプレ名 | 式 | 用途 |
|---|---|---|
| `H_pass` | `Σ 中計` | カテゴリ別利益率モード時（中計で利益率消化済み） |
| `H_overall` | `Σ 中計 × (1 + 利益率_全体)` | 全体一括利益率モード |
| `H_lookup` | `Σ 中計 × (1 + 利益率表[Σ 中計])` | LOOKUP 利益率モード |

→ **利益率モードの違い = G/H のテンプレート選択の違い**

## 4. 統一パターン（明細を作る UI フロー）

すべての見積カテゴリで **同じフロー**:

```
[+ 追加] ボタン
  ↓
選択肢のプルダウン (1 階層目)
  ↓
選択肢のプルダウン (2 階層目)  ← 必要なら
  ↓
... (葉に到達するまで動的に降りる)
  ↓
変数入力エリア（計算式が要求する変数のフォーム）
  ↓
小計が確定
```

**動的階層**: 階層の深さは枝ごとに違う。**葉に到達した瞬間に F（計算式テンプレート）が一意に決まる**。

例:

```
材料費
├─ 鋳造材 ────────────────【F: 鋳造材費】 ← 1 階層で決まる
├─ 板金材
│    ├─ SS400 ──────────【F: SS400 重量計算】 ← 2 階層
│    └─ SUS304 ─────────【F: SUS304 重量計算】
└─ 角材
     ├─ アルミ
     │   ├─ 重量ベース ─【F_A: アルミ重量計算】 ← 3 階層
     │   └─ 体積ベース ─【F_B: アルミ体積計算】
     └─ 鉄
         └─ 重量ベース ─【F_C: 鉄重量計算】
```

## 5. 選択肢の木のノードは 2 つの役割を持つ

各ノードは役割を **A 必須 + D は葉のみ必須** で持つ:

| 役割 | 何をするか | 例 |
|---|---|---|
| **A. ナビゲーション + 値の保有** | 次の選択肢を絞る + ノードマスタ値（属性付き）を指す | 「板金材」「SS400」「特急」 |
| **D. 計算式の確定**（葉のみ） | F テンプレートを決定 | 「重量ベース」 |

中間ノードは A のみ、葉は A + D。

### ノードマスタ値が属性を運ぶ

役割 A の本体は **ノードマスタ値** (`node_master_values.values JSONB`)。各値が `attribute_schema` に従った属性を保持する。

例: 「材料種別」マスタの値「SS400」は `{title: "SS400", category: "鉄鋼", density: 7.85, unit_price: 380}` を持つ。これらの属性が経路上に積み上がり、PATH 変数として計算式から参照される（→ §6）。

### ノード設定の例

```
材料費
├─ 板金材        ← ノードマスタ「材料項目」の値「板金材」
│   ├─ SS400    ← ノードマスタ「材料種別」の値「SS400」(density, unit_price 属性付き)
│   │   ├─ 重量ベース [葉]  formula = W × ρ × P × Y
│   │   └─ 体積ベース [葉]  formula = V × ρ × P × Y
│   └─ SUS304   ← 同上、属性が違うだけ
└─ 角材
```

外注費の例（取引先・係数も全てノードマスタ値の属性として持つ）:

```
外注費
├─ メッキ                ← ノードマスタ「加工方法」の値
│   ├─ ◯◯メッキ工業     ← 取引先ノードマスタ値（rank, unit_price 属性）
│   │   ├─ 通常 [葉]     ← 「処理区分」マスタ値（係数=1.0 属性）
│   │   └─ 特急 [葉]     ←  同上 (係数=1.5 属性)
│   └─ △△鉱業
└─ 焼入れ
```

→ 「マスタ参照を別建てで持つ（旧役割 B）」「固定変数を別建てで持つ（旧役割 C）」は廃止。**全部ノードマスタ値の属性に一本化** した。

## 6. 変数の値の出どころ（3 ソース）

```
変数の値 = MANUAL  → ユーザーが画面で入れた値
        LOOKUP   → 表から検索した値
        PATH     → 選択経路上のノードマスタ値の属性から取得
```

PATH 変数は **`variable_definitions.path_key`** で取り出すキーを指定する。選択肢の木を辿る過程で:

- 各ノードが指す `node_master_values.values JSONB` を順番にマージ（深い方が勝つ）
- PATH 変数は変数定義の `path_key` でその辞書からピンポイントに値を取り出す

例: 変数 `rho`（path_key = `density`）は、経路上のいずれかのノード値が `{density: 7.85, ...}` を持っていればそこから 7.85 が解決される。

## 7. LOOKUP テーブル（参照表）

軸の組合せで単一値を引く。3 種類のマッチ方式:

| 方式 | 意味 | 行側のキー |
|---|---|---|
| **EXACT** | 完全一致 | `{axis}` |
| **RANGE** | 範囲（半開区間 `[min, max)`） | `{axis}_min`, `{axis}_max` |
| **PREFIX** | 前方一致 | `{axis}_prefix` |

### バージョン管理

- LOOKUP は **テーブル → バージョン → 行データ** の 3 層
- 新バージョンを作ると旧バージョンは自動で `ARCHIVED`
- 過去見積もりは保存時点のバージョンで再現される

### 線形補間（RANGE 軸のみ）

区間ごとに `CONSTANT`（補間なし、値固定）か `LINEAR`（線形補間）を選べる:

```
LINEAR 補間値 = 下端値 + (上端値 - 下端値) × (軸の値 - 区間下端) / (区間上端 - 区間下端)
```

`return_value`（下端値）と `return_value_max`（上端値、NULL なら CONSTANT）の 2 列で表現。

## 8. 利益率モード（3 種）

### モード A: カテゴリ別利益率

```
G_c(小計達) = (Σ 小計 + 運賃 if c=O) × (1 + 利益率_c)
H(中計達)  = 丸め( Σ 中計 )
```

材料費 10%、工程費 30% のようにカテゴリで率を変える。

### モード B: 全体一括利益率

```
G_c(小計達) = Σ 小計 + 運賃 if c=O      ← 利益率なし
H(中計達)  = 丸め( Σ 中計 × (1 + 利益率_全体) )
```

### モード C: LOOKUP 利益率

```
G_c(小計達) = Σ 小計 + 運賃 if c=O      ← 利益率なし
H(中計達)  = 丸め( Σ 中計 × (1 + 利益率表[Σ 中計]) )
```

金額帯・数量帯で利益率を段階的に変える運用。

### 循環防止ルール

利益率 LOOKUP の軸には **「合計」を選べない**（売価で利益率が決まる ↔ 利益率で売価が決まる、の循環を避ける）。軸として許可する量:

- 原価合計（中計の合計）
- カテゴリ小計（中計）
- 数量
- その他、利益率の結果に依存しない量

## 9. 上書き 3 レバー（柔軟な調整）

「計算は参考、決めるのは人」が設計思想。**小計・中計・合計の 3 階層** すべてで上書き可能。

```
最終値 = もし完全上書きあり → 上書き値
        そうでなければ    → ( 計算値 × 倍率 ) + 加算
```

3 つのレバー:

| レバー | 例 | 用途 |
|---|---|---|
| **完全上書き** | `final = 5500` | 計算は無視、手動で値を決める |
| **倍率** | `final = calc × 1.1` | 10% 上乗せ、5% 値引き等 |
| **加算** | `final = calc + 500` | 調整費、特別費用 |

組み合わせ可能: `final = calc × 1.1 + 500`

デフォルト: 倍率 = 1.0, 加算 = 0, 完全上書き = なし。

UI 上は **ペンマーク** ボタンを押すとモーダルで編集できる。

## 10. 数量バリエーション（ロット）

1 つの見積もりに **複数の数量パターン** を持てる:

```
見積もり (Item: シャフト A-100)
  ├ 数量バリエーション q=100   → 合計 138,000 円 / 1,380 円・個
  ├ 数量バリエーション q=500   → 合計 650,000 円 / 1,300 円・個
  └ 数量バリエーション q=1000  → 合計 1,200,000 円 / 1,200 円・個
```

各バリエーションが独立して上の式を回し、独立した合計を出す。**段階見積もりの提示** に直接対応。

数量で変わるもの:
- 材料費 → 数量に比例
- 加工費 → 段取り費は固定、加工時間は数量に比例
- 利益率 → ボリュームディスカウントで段階的に下がる（LOOKUP）

## 11. 全体を 1 本の式で

```
合計 = 丸め(
  Σ_見積カテゴリ
    [ G_c (
        小計達 = [ adjust_F( F( 変数達 ) ) for 各明細 ],
        運賃達 = [外注のみ]
      )
    ] -- ↑ 中計のリスト
  → adjust_G で各中計を補正
  → H( 中計達 ) で集約
  → adjust_H で全体補正
  → 丸め
)
```

シンプルに言えば:

```
小計 = adjust_F( F( 変数達 ) )
中計 = adjust_G( G_c( 小計達 ) )
合計 = 丸め( adjust_H( H( 中計達 ) ) )
```

各層に「テンプレ計算 + 上書き 3 レバー」が入っている。

---

## 12. 押さえるべき骨組み

```
┌──────────────────────────────────────────┐
│ 合計 = 丸め( Σ ( 中計 × (1 + 利益率) ) )    │
└──────────────────────────────────────────┘
              ▲                    ▲
              │                    │
              │            G_margin or H_overall or H_lookup
              │
        Σ_明細 小計（+ 外注の運賃）
              ▲
              │
        adjust（倍率・加算・完全上書き）
              ▲
              │
        F = 計算式テンプレート( 変数達 )
              ▲
              │
        手入力 / マスタ / LOOKUP / パス変数
              ▲
              │
        [選択肢の木 (N 階層)] → 葉で F が一意に決定
```

「Σ の入れ子 × 利益率倍率 × 丸め」が骨組み。残りは枝葉:
- 上書き 3 レバー（小計・中計・合計）
- 外注費の運賃加算
- LOOKUP の線形補間
- 利益率の LOOKUP 化
- 数量バリエーション
- 選択肢の木のパス変数
