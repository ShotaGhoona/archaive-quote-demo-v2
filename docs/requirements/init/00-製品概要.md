# 00. 製品概要

## 何を作るか

**Archaive-Quote** は、製造業向けの汎用原価見積モジュールのデモ。図面 1 枚ごとに「材料費・工程費・外注費・購入品・諸経費」の 5 カテゴリで原価を積み上げ、利益率を乗せて売価を出す。

会社ごとに違う見積もりロジック（材料の分類体系、計算式、単価表、利益率の運用）を、共通骨格を維持したまま **設定で吸収できる** ことを売りにする。

## なぜ作るか

製造業の見積もりは、これまで Excel テンプレで属人的に運用されてきた。問題:

- 担当が変わると基準がブレる
- 単価更新の漏れ
- 過去の見積もりを再現できない（マスタが変わっているから）
- 掛け率を金額帯で変える運用が IF 文で破綻する
- 計算式が頭の中（ベテランが辞めると失われる）

これを **マスタ + 計算式 + ルックアップ** の 3 層で形式化する。会社差異は「自由設計部分」で吸収。

## 誰のためか

3 つのペルソナ:

| ペルソナ | 主な操作 |
|---|---|
| 営業担当（佐藤さん） | 見積もりを作って顧客に出す |
| 設計／積算担当（田中さん） | 計算式とマスタを保守、複雑な形状の積算 |
| 管理者（鈴木さん） | 単価更新、運用設定、監査対応 |

## スコープ

### v2 デモで作るもの

- 1 つの Item（製品）に対して見積もりを作成・編集できる
- 5 カテゴリの明細を「N 階層の選択 + 変数入力」の統一パターンで積み上げる
- 計算式テンプレート・変数定義・LOOKUP テーブルを設定画面で編集できる
- 利益率モード 3 種（カテゴリ別 / 全体一括 / LOOKUP）を切り替えられる
- 小計・中計・合計のいずれでも「上書き 3 レバー」（完全上書き / 倍率 / 加算）で調整できる
- 数量バリエーション（100 個・500 個・1000 個 など）の段階見積もりに対応

### スコープ外（v3 以降）

- AI チャット・図面 OCR
- メール送信
- 発注確定・サプライヤーリコメンド
- マルチテナント（デモは単一会社）
- 認証 UI（Supabase Auth は導入するが、運用 UI 簡略化）
- PDF / Excel 出力（後付け）
- BOM 階層展開
- 実績との突き合わせ

## ビジネス価値

| 価値       | 内容                          |
| -------- | --------------------------- |
| 属人化解消    | 計算式・マスタが社内資産になる             |
| 再現性      | 過去見積もりを開いても保存時点の値で再計算される    |
| 柔軟性      | 会社ごとの運用差を設定で吸収。テンプレート流用しやすい |
| 単価更新の即反映 | マスタ更新で新規見積もりに即時反映           |
| 段階見積もり対応 | 100 個と 500 個の単価を 1 回で出せる    |
| 営業の判断余地  | 上書き 3 レバーで現場判断を反映           |

## 設計戦略のキモ: ハードコード vs 自由化

製造業の見積もりは「会社ごとにバラバラ」が根本問題。これに対する答え:

| 部分 | ハードコード（共通骨格） | 自由化（会社ごと設計） |
|---|---|---|
| カテゴリ枠組み | ✅ 5 種固定（ON/OFF のみ可） | |
| 統一パターン（選択 + 変数入力） | ✅ | |
| 合計／中計／小計の階層 | ✅ | |
| 利益率と丸めの仕組み | ✅ | |
| 選択肢の木の構造と深さ | | ✅ |
| 計算式テンプレート | | ✅ |
| 変数定義 | | ✅ |
| LOOKUP テーブル | | ✅ |
| 利益率モードと値 | | ✅ |

→ **共通骨格は固定、肉付けは自由設計**。これが汎用化の本質。

## デモのゴール

claude-code が poc-template をクローンして、本資料に従って実装すれば:

1. ある Item の画面で図面プレビュー＋見積もりパネルが動く
2. 設定画面でマスタ・計算式・LOOKUP が編集できる
3. 1 つの引合いで段階見積もり（数量別）が出せる
4. 利益率モードを切り替えても合計が正しく出る
5. 上書き 3 レバーで明細・中計・合計を調整できる

→ JAM／TAMAYAMA に「これが汎用見積もりエンジンです」と提示できる状態。
