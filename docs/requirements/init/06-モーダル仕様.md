# 06. モーダル仕様

shadcn/ui の `<Dialog>` をベースにする。フォームは `react-hook-form + Zod`。

---

## 1. 上書き編集モーダル（ペンマーク押下）

### 用途

小計・中計・合計の各レベルで共通。3 レバー（完全上書き / 倍率 / 加算）を編集する。

### モックアップ

```
┌─────────────────────────────────────┐
│ 小計の上書き編集                     │
│   計算結果: 5,000 円                  │
├─────────────────────────────────────┤
│                                     │
│ ☐ 完全上書き              [        ] │
│ ☐ 倍率                  × [        ] │
│ ☐ 加算                  + [        ] │
│                                     │
│ プレビュー: 計算 5,000 → 採用 5,500 円 │
│                                     │
│              [キャンセル]  [保存]      │
└─────────────────────────────────────┘
```

### Zod スキーマ

```typescript
const adjustSchema = z.object({
  override_value: z.number().nullable(),
  multiplier: z.number().min(0).default(1.0),
  adjustment: z.number().default(0),
});
```

### 振る舞い

- 各チェックボックスを ON にすると、その入力欄が活性化
- 完全上書きを ON にすると、倍率・加算は無効（グレーアウト）
- プレビュー欄に「計算 5000 → 採用 5500 円」のようにリアルタイム表示
- [保存] でサーバーに保存して再計算 → 親画面の小計が更新

### 開かれる場所

- 明細の小計（5,000 円 ✏）
- カテゴリの中計（12,000 円 ✏）
- 合計エリア（30,000 円 ✏）

### コンポーネント名

`<AdjustModal level="subtotal" | "category" | "total" id={...} />`

## 2. 選択肢ノード追加・編集モーダル（カラムビュー組み込み）

### 用途

選択肢の木のノードを追加・編集する。実装ではカラムビュー（画面 03 §5）に直接組み込まれており、列ごとに「どのノードマスタを使うか」「葉として計算式を紐付けるか」を選ぶ流れになっている。

### モックアップ

```
┌────────────────────────────────────────────┐
│ ノード追加: 角材 配下                        │
├────────────────────────────────────────────┤
│ 親ノード   [角材]                            │
│                                            │
│ ノードマスタ（種類） [材料種別 ▼]            │
│ 値（既存から選ぶ）   [SS400 ▼]              │
│   または [+ 新しい値を作成]                   │
│                                            │
│ 葉として確定する                              │
│ ☐ このノードを葉にする（計算式に紐づく）      │
│   計算式テンプレート [SS400 重量計算 ▼]     │
│   または [+ 新規計算式]                       │
│                                            │
│              [キャンセル]  [保存]            │
└────────────────────────────────────────────┘
```

### 振る舞い

- 親ノードのカテゴリを継承
- 「葉にする」を ON にしたら計算式テンプレートを必ず指定（葉は子を持てない）
- 値はノードマスタ値から選ぶか、その場で新規作成（属性入力は値追加モーダル）
- マスタ参照や固定変数の概念は廃止 — ノードマスタ値の属性に統一されている

### Zod スキーマ

```typescript
const nodeSchema = z.object({
  parent_id: z.string().uuid().nullable(),
  category: z.enum(['MATERIAL', 'IN_HOUSE', 'OUTSOURCE', 'PURCHASE', 'EXPENSE']),
  label: z.string().min(1),
  node_master_value_id: z.string().uuid().nullable(),
  formula_template_id: z.string().uuid().nullable(),
});
```

## 3. 計算式テンプレート編集モーダル（電卓 UI）

### 配置場所

設定画面 → 見積もり設定 → **「その他マスタ」タブ** → 計算式セクション。
（カテゴリ別設定からは独立した、共通マスタとして管理される）

### モックアップ

```
┌──────────────────────────────────────────────────────────────┐
│ 計算式テンプレート編集                                        │
├──────────────────────────────────────────────────────────────┤
│ 名前         [SS400 重量計算]                                 │
│ 種別         ● F (小計)  ○ G (中計)  ○ H (合計)             │
│ 対象カテゴリ [材料費 ▼]                                       │
│                                                              │
│ ┌─ プレビュー ───────────────────────────────────────────┐  │
│ │ 板厚 × 長さ × 幅 × 比重 ÷ 1000 × 単価 × 歩留まり      │  │
│ │ （変数コードはラベルで表示、× ÷ もユーザー向けに整形）│  │
│ └────────────────────────────────────────────────────────┘  │
│                                                              │
│ ┌─ 使用する変数（クリックで挿入） ──────────────────────┐    │
│ │ [板厚] [長さ] [幅] [比重] [単価] [歩留まり]           │    │
│ │ [+ 新規変数]                                          │    │
│ └──────────────────────────────────────────────────────┘    │
│                                                              │
│ ┌─ 電卓パッド ──────────────────────────────────────────┐    │
│ │  7  8  9  ÷                                           │    │
│ │  4  5  6  ×                                           │    │
│ │  1  2  3  -                                           │    │
│ │  0  .  ^  +              [(] [)] [//] [pow]           │    │
│ │                                                        │    │
│ │  [⌫ Backspace]  [↺ Reset]                             │    │
│ └──────────────────────────────────────────────────────┘    │
│                                                              │
│              [キャンセル]  [保存]                              │
└──────────────────────────────────────────────────────────────┘
```

### 振る舞い

- **電卓 UI**: 数字・演算子はパッドのボタンで入力。直接タイピングも可能
- **変数の挿入**: 上部の変数チップをクリックすると現在のカーソル位置に変数コードが挿入される
- **プレビュー**: 変数コードはラベル（日本語）に置き換えて表示、`*` `/` は `×` `÷` に整形
- **新規変数**: 「+ 新規変数」から `VariableDialog` を開き、その場で変数定義を作って挿入できる
- 保存時にバックエンドで構文 + 未定義変数チェック

### F/G/H の選択

- F の場合: 対象カテゴリ必須
- G の場合: ビルトイン (`G_pass` / `G_outsource` / `G_margin`) or カスタム
- H の場合: ビルトイン (`H_pass` / `H_overall` / `H_lookup`) or カスタム

## 4. 変数定義編集モーダル

### モックアップ

```
┌──────────────────────────────────────────────────┐
│ 変数定義 編集                                     │
├──────────────────────────────────────────────────┤
│ コード     [d            ]                        │
│ ラベル     [直径          ]                       │
│ 型         [NUMBER ▼]                            │
│ 単位       [mm ▼]                                │
│ 必須       ☑                                     │
│ 説明       [...                          ]        │
│                                                  │
│ ソース     ● MANUAL（手入力）                     │
│           ○ LOOKUP                               │
│           ○ PATH（経路上のノード値属性から）       │
│                                                  │
│ ── PATH 時 ──                                     │
│ path_key   [density       ]                       │
│   （経路ノード値 JSON のキー名。空ならコードを使う）│
│                                                  │
│ ── LOOKUP 時 ──                                   │
│ テーブル     [歩留まり表 ▼]                       │
│                                                  │
│              [キャンセル]  [保存]                  │
└──────────────────────────────────────────────────┘
```

### 振る舞い

- ソースを切り替えるとそれぞれ専用フィールドが出現
- SELECT 型を選ぶと options 編集 UI が出る
- 単位は単位マスタから選ぶ（プルダウン）

## 5. ノードマスタ編集モーダル

### モックアップ

```
┌────────────────────────────────────────────┐
│ ノードマスタ編集: 材料種別                  │
├────────────────────────────────────────────┤
│ 種類名 [材料種別           ]                │
│ 説明   [材料の具体的な種類を表す]           │
│                                            │
│ 持つ属性                                    │
│  ┌─────────────────────────────────────┐  │
│  │ key      | label    | type   | 必須 │  │
│  │ title    | タイトル  | STRING | ✓    │ [↑↓🗑]
│  │ category | カテゴリ  | STRING |      │ [↑↓🗑]
│  │ 比重     | 比重      | NUMBER |      │ [↑↓🗑]
│  │ 単価     | 単価      | NUMBER |      │ [↑↓🗑]
│  │ [+ 属性追加]                          │  │
│  └─────────────────────────────────────┘  │
│                                            │
│ 登録された値                                │
│  ┌─────────────────────────────────────┐  │
│  │ title    | category | 比重 | 単価   │  │
│  │ SS400    | 鉄鋼     | 7.85 | 380   │ [✏🗑]
│  │ SUS304   | ステンレス| 7.93 | 420   │ [✏🗑]
│  │ A5052    | アルミ   | 2.68 | 600   │ [✏🗑]
│  │ [+ 値を追加]                          │  │
│  └─────────────────────────────────────┘  │
│                                            │
│         [キャンセル]  [保存]                │
└────────────────────────────────────────────┘
```

### 振る舞い

- 属性スキーマと値を同じモーダル内で管理（タブ分けでも可）
- 属性追加 → 既存の値レコードに該当キーが追加される（NULL で初期化）
- 値の追加 → 別モーダル（簡易フォーム）

## 6. LOOKUP 行追加モーダル

### モックアップ

```
┌────────────────────────────────────────────┐
│ LOOKUP 行追加: 歩留まり表                   │
├────────────────────────────────────────────┤
│ 軸 1 (material_code, EXACT)                │
│   値 [SS400         ]                      │
│                                            │
│ 軸 2 (直径, RANGE)                          │
│   下端 [30          ]                      │
│   上端 [60          ]                      │
│                                            │
│ ── 戻り値 ──                                │
│ 値（下端）  [1.10    ]                      │
│ 値（上端）  [1.20    ]  (LINEAR 時のみ)     │
│ 補間種別     ○ CONSTANT  ● LINEAR           │
│                                            │
│         [キャンセル]  [保存]                │
└────────────────────────────────────────────┘
```

### 振る舞い

- 軸定義に従ってフォームが動的生成される
- EXACT 軸は単一値、RANGE 軸は下端・上端、PREFIX 軸は文字列入力
- 補間種別が LINEAR のとき「値（上端）」が必須
- 行追加時、既存行と重複していないかバリデーション（複数マッチを避ける）

## 7. Item 新規作成・編集モーダル

### モックアップ

```
┌────────────────────────────────────────┐
│ Item 新規作成                           │
├────────────────────────────────────────┤
│ 品番       [ABC-001          ]          │
│ 品名       [シャフト A-100    ]          │
│ 取引先     [★★製作所 ▼]                 │
│                                        │
│ 図面       [📤 ファイルを選択]            │
│            (PDF / 画像、Storage に保存)   │
│                                        │
│ 備考       [...                  ]      │
│                                        │
│         [キャンセル]  [保存]             │
└────────────────────────────────────────┘
```

### 振る舞い

- 品番は必須・ユニーク
- 図面アップロードは Supabase Storage に保存し、URL を `items.drawing_url` に
- 保存後 `/items/:id` に自動遷移して見積もり作成へ

## 9. 確認ダイアログ（AlertDialog）

削除など破壊的操作はすべて `<AlertDialog>` で確認:

```
┌────────────────────────────────────────┐
│ ⚠ この明細を削除しますか？               │
│                                        │
│ 計算結果も含めて完全に削除されます。     │
│                                        │
│         [キャンセル]  [削除]             │
└────────────────────────────────────────┘
```

---

## 10. 共通の振る舞いルール

- **保存中はボタンを非活性 + スピナー**
- **エラー時は sonner の toast.error**
- **成功時は toast.success**
- **モーダルは ESC キーで閉じる**
- **モーダル外をクリックで閉じる（破壊的でない場合）**
